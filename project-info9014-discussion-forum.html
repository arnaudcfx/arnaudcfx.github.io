Knowledge Representation and Reasoning – Ontology Project: Discussion Forum

OWL 2 RL RDF R2RML MySQL Protégé Python (rdflib)

Project Overview

This project focuses on developing an ontology for online discussion forums as part of the Knowledge Representation and Reasoning course. The goal was to model how forums, threads, posts, users, and relationships interact in an online community, and then map this relational data to RDF using R2RML to build a knowledge graph. The ontology enables semantic querying and reasoning on forum data.

Information System Design

The project began by creating a relational database for the forum containing entities such as users, posts, threads, categories, and roles. Constraints ensured data consistency and prevented duplicates. This schema served as the foundation for the ontology mapping.

Ontology Design and Implementation

  • Profile chosen: OWL 2 RL for scalable reasoning with adequate expressiveness.
  • Classes include: Forum, Thread, Post, User, Profile, Role, Tag, Category.
  • Object properties like hasProfile, isSubscribedToForum, and hasTag model relationships.
  • Inheritance introduced with a SuperUser class extending User for admin/moderator roles.
  • Ontology built and validated using Protégé and checked for OWL 2 RL compliance.

Mapping Relational Database to RDF

  • Used R2RML mappings to generate RDF triples from relational tables.
  • Each table was mapped to an RDF class (e.g., User → forum:User).
  • Predicate-object mappings aligned SQL columns with ontology properties.
  • Reference object maps established relationships via primary and foreign keys.
  • Consistency checked in Protégé with the Ontop plugin and live MySQL database connection.

Knowledge Graph and Queries

The resulting RDF graph was imported into Python using rdflib for querying. Non-trivial SPARQL queries were designed to analyze user behavior, popular tags, post counts, and forum participation patterns.

Demonstration and Visualization

  • Ontology visualized using WebVOWL for clear class-property relationships.
  • Knowledge graph visualized using Graphviz to illustrate user–post–forum interconnections.
  • Planned web-based demonstrator: an application allowing semantic exploration and visualization of forum activity through graph queries.

Conclusions

Through this project, we gained practical experience in ontology engineering, knowledge graph design, and database-to-RDF mapping. We explored how logical modeling and semantic technologies can represent social interaction data and support intelligent querying of complex relational systems.
