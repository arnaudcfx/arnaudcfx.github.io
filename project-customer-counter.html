Automated Customer Counter (IR Barrier)

Embedded C / Assembly Microcontroller (PIC16F1789) Electronics Makefile

1. Introduction & Project Idea

The project aims to develop an automated customer counter for shops using an infrared (IR) system. This system helps regulate the number of customers entering a store, addressing sanitary restrictions by replacing inefficient manual counting or trolley-based methods with reliable technology.

The solution consists of two IR-emitting circuits and a central receiving circuit, enabling real-time counting of customers entering and leaving the premises.

2. System Overview & Operation

  • IR Emitters: Two circuits at the entrance and exit, each generating a 38kHz modulated IR signal.
  • IR Receiver (Central Unit): Equipped with two IR receivers (left/right), four configuration buttons, and two 7-segment displays to show the remaining allowed customers.
  • Workflow:
    • Manager sets max allowed customers with buttons; confirmed via displays.
    • On entry: counter decrements, displays update.
    • On exit: counter increments, displays update.
    • Mode switching and reset possible via dedicated buttons.
  • IR detection is based on voltage changes from the receivers (0V = beam detected, ~5V = beam interrupted).

3. Hardware Components

Receiver Circuit:
  • PIC16F1789 microcontroller – core logic unit
  • 2x TSOP58438 IR receivers – robust, immune to noise
  • 2x LS0565SRWK 7-segment displays – red, common cathode
  • 4x Pushbuttons – for configuration and reset
  • L7805CV voltage regulator, resistors, capacitors, LEDs, battery holder, etc.
Emitter Circuits (x2):
  • TSHF5410 IR emitter LEDs
  • TLC555IP timer (generates 38kHz modulation)
  • Supporting discrete components (resistors, capacitors, diodes, battery, etc.)

Component selection prioritized compatibility, price, and availability. The receiver uses modulated IR for ambient light immunity, as advised by experts.

  • Receiver Circuit: Central logic, display, and IR detection; supports setup and operation modes with visual feedback (status LED).

  • Emitter Circuits: Each placed at entry/exit, continuously generating IR beams for the barrier.
  • User Interface: Four buttons allow the user to reset, select, increment, or decrement the maximum customer count.

5. Software Architecture

  • Round Robin with Interrupts: Chosen for simplicity and responsiveness; interrupts handle barrier and button events. No task prioritization needed.
  • Modes:
    • Configuration: Set maximum allowed customers; button events handled via interrupts.
    • Operation: Entry/exit detection and count update; display refresh outside ISRs to keep routines short.
  • Display Updates: Performed only on value changes for efficiency.

7. Results & Learnings

  • Reliable real-time counting of customers; no missed events in lab tests.
  • Successfully handled electrical noise thanks to modulated IR and TSOP receivers.
  • Learned the importance of correct sensor selection and robust circuit design.
  • Achieved a practical solution with affordable microcontroller hardware.
  • Improved understanding of embedded interrupts, digital interfacing, and debugging.
