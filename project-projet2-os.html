Hacking the Kernel

C Linux Kernel Memory Management

Project Description

Project 2 focuses on hacking the Linux kernel to add custom system calls for memory page analysis. The project required a deep dive into kernel data structures and process memory management, with the goal of exposing memory usage information via new syscalls.

  • getnbpages(): Given a process name, this syscall fetches all processes with that name (using addName()). For each process, it collects their memory management structures (mm) via addMm(). For each memory mapping (vm_area_struct), it calculates the number of pages in each region by subtracting start and end addresses and dividing by the page size (4KB). Page table entries (ptes) are retrieved and indexed using addPte(). The total page count is accumulated with addPages().
  • getreadpages(): Fetches a bitmap from user space and sets each bit according to the pte indices previously gathered. The updated bitmap is then copied back to user space using copy_to_user(). All intermediate kernel structures (mms, ptes, Processes) are cleaned up after execution.

Kernel Memory Management Overview

  • Each process is represented by a task_struct, which holds metadata like PID, name, and flags. The mm_struct field manages process memory.
  • mm_struct tracks all virtual memory areas (VMAs) and the page table root (pgd).
  • Each VMA defines a contiguous region of virtual addresses (heap, stack, etc.), with start/end addresses and access flags.
  • VMAs are split into fixed-size pages, each mapped to a physical RAM frame by the MMU. Virtual pages are indexed via page tables: pgd → pud → pmd → pte.
  • The page number indexes into the page table hierarchy to locate the physical address. Each upper-level table points to a block of lower-level tables, with pgd as the top level.
