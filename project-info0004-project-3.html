Drunk Run 2.5D – Object-Oriented Programming Project

Project Summary

Drunk Run 2.5D is a C++ project developed as part of the INFO0004 – Object-Oriented Programming course. The goal was to design a 2.5D game where the player controls a character named Cymi who runs through a perspective-based environment filled with obstacles. The project focuses on object-oriented design, real-time graphics, and game loop management in C++.

Architecture Overview

The central class Game manages the window creation, rendering loop, score display, and global elements. It contains a vector of obstacles (e.g., Rubbish, Person, Crate, Barrier) and a Cymi object representing the player. Each obstacle inherits from the Box class, itself derived from the base class GameEntity, enabling 2.5D projection behavior and interaction with Cymi’s movements.

This structure allows for strong encapsulation and reuse: obstacles observe Cymi’s state to adapt their behavior, while Cymi manages user input, random movement, and path rendering.

Key Features & Mechanics

  • Perspective Rendering: The game computes a realistic field of view using trigonometric relations to match the player’s vanishing point.
  • Obstacle Generation: Obstacles are generated using uniform random distributions across x and z axes to create varied gameplay.
  • Optimized Rendering: Obstacles are sorted by z-axis depth for correct rendering order and scaling to screen coordinates.
  • Frame Rate Control: The system enforces a constant 30 FPS by adjusting loop delays dynamically based on update times.
  • Player Controls: Cymi moves automatically forward and can perform side jumps with randomized movement patterns every two seconds.

Technical Design

  • GameEntity: Base class for all visible game objects, managing position, drawing, and updates.
  • Box: Derived class implementing 2.5D projection logic for obstacle representation.
  • Rubbish, Person, Crate, Barrier: Specific obstacle classes extending Box to define unique interactions.
  • Cymi: Player character handling movement, collision detection, and random behavior.
  • Game: Central loop controlling updates, rendering, and time management.

Object-Oriented Design Principles

The project strongly emphasizes inheritance and polymorphism for flexibility in game entities. The choice to avoid the Prototype pattern in favor of polymorphism reduces redundancy and simplifies object management. Each class defines clear responsibilities.

Technologies & Tools Used

  • C++ – Core language for all classes and game logic.
  • Makefile – Build automation and dependency management.
  • OOP Patterns – Inheritance, encapsulation, and polymorphism.
  • Mathematics & Trigonometry – Used for perspective calculations and coordinate transformations.
  • SFML / SDL : Potential libraries for graphics and event handling.

In summary, Drunk Run 2.5D showw the integration of geometric computation, event systems, and object-oriented programming within an interactive C++ game environment.
