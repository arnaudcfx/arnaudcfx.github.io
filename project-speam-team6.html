VeGApp – My Autonomous Vegetable Garden App

Flutter Java Spring Boot PostgreSQL OpenShift GitLab CI/CD Docker

Project Overview

VeGApp (Vegetable Garden Application) transforms traditional gardening into a smart, connected experience. It allows users to manage and monitor their vegetable gardens via mobile and web applications. The system integrates IoT-inspired concepts with database synchronization and a continuous deployment pipeline, creating a seamless, automated ecosystem for gardeners.

Architecture Overview

  • Frontend: Developed with Flutter following the MVC architecture, with models for data management, views for user interfaces, and controllers for input handling and synchronization.
  • Backend: Java Spring Boot server using a REST API architecture, handling synchronization, authentication, and data persistence.
  • Database: PostgreSQL database for structured data storage, accessible both locally and through OpenShift deployment.
  • CI/CD: GitLab pipelines automate build, test, and deployment with Docker and OpenShift integration.

Testing and Methodology

The project follows a hybrid Model-Driven + Agile/Scrum methodology. Model refinement and iteration through sprints ensured alignment with user needs. Testing was approached as a continuous process — from unit tests in Flutter and Java to integration and pipeline-level validation within GitLab.

  • Frontend Testing: Automated tests run in the CI pipeline before APK builds.
  • Backend Testing: API validation through Python scripts simulating network requests and offline scenarios.
  • Synchronization Testing: Verified database and API resilience during connection loss and recovery.

Key Technologies

  • Flutter: For cross-platform mobile application development.
  • Spring Boot: For the Java-based backend services.
  • PostgreSQL: For persistent, relational data storage.
  • GitLab CI/CD: For automated build and deployment workflows.
  • Docker & OpenShift: For containerization and deployment management.

Code Architecture

Frontend

The Flutter frontend adopts the MVC pattern. Models represent entities like gardens, plants, and plots. Views correspond to application screens, and Controllers handle logic, synchronization, and user interaction. This modular design supports maintainability and scalability.

Backend

The Spring Boot backend manages user authentication, synchronization logic, and communication with the PostgreSQL database. Hibernate provides ORM mapping between Java entities and SQL tables. The server employs token-based authentication and secure HTTPS communication with a self-signed certificate.

Website

The website complements the mobile app with a responsive web interface for managing gardens, permissions, and user accounts. It includes multiple HTML pages styled with CSS for distinct features such as login, signup, and garden management.
