HeroTrainer – Object-Oriented Mobile RPG Game

Project Summary

HeroTrainer is a role-playing game (RPG) developed as part of the Object-Oriented Programming on Mobile Devices course. It is a mobile and desktop-friendly game where users can train, equip, and battle with heroes of different classes. Each hero evolves over time through combat, item crafting, and trading with merchants. The project was built to demonstrate strong object-oriented design principles and a modular architecture for scalable game development.

Application Overview

  • Loading Screen: Entry point where the player starts the game.
  • Select Screen: Players choose among nine hero classes, each with unique abilities.
  • Home Screen: Central hub for saving progress and navigating between screens.
  • Craft Screen: Allows merging of compatible items to create stronger ones.
  • Merchant Screen: Shop where players can buy or sell items from their inventory.
  • Hero Screen: Displays hero stats, equipment, and inventory.
  • Combat Screen: Turn-based battle interface where players fight monsters using attacks, potions, or spells.

Technical Design

The game’s architecture is divided into four main layers, ensuring clean separation of responsibilities and easy maintainability:

  • Data Entities: Contain data structures and object models such as heroes, monsters, items, and spells.
  • Business Logic: Implements gameplay mechanics and communicates with UI through the Provider pattern.
  • User Interface: Developed using Flutter/Dart for a responsive, animated mobile interface.
  • Services: Manage saving, loading, and other device-related operations.

Object-Oriented Structure

The entire codebase follows object-oriented principles. The core class Character serves as an abstract base, defining shared attributes and methods for both heroes and enemies. Derived classes extend functionality through polymorphism, implementing unique combat behaviors, spells, and level-up logic. This architecture enables scalability and simplifies debugging and maintenance.

Combat & Items System

The turn-based combat system uses event-driven states to handle player and enemy actions. Each character can perform attacks, cast spells, or use items within time constraints. Effects like poison, armor boosts, and burn damage can persist over multiple turns, creating complex strategies. Items such as weapons, armor, and potions are procedurally generated based on hero level and rarity, allowing dynamic gameplay and replayability.

Limitations & Future Improvements

  • Multiplayer mode was planned but not implemented.
  • Automatic game saving and expanded crafting options (alchemy, forging) could enhance experience.
  • Future updates could include unique hero equipment, improved balancing, and additional sound effects.

Technologies & Tools Used

  • Dart & Flutter – Cross-platform mobile app framework for UI.
  • Java – Backend logic and data model implementation.
  • Makefile – For build automation and dependency management.
  • OOP Patterns – Abstract classes, inheritance, and polymorphism for game logic.
  • Git / GitHub – Version control and collaborative development.

Key Takeaways

HeroTrainer successfully demonstrates the use of object-oriented programming and mobile game architecture. The project reinforced skills in software design, event-driven systems, and collaborative coding while producing a fully playable, animated RPG experience.

In summary, this project showcases the ability to design modular, object-oriented mobile applications integrating backend logic, UI, and gameplay elements within a cohesive system.
